Wat maakt een website snel? Core Web Vitals uitgelegd

LCP, INP en CLS in begrijpelijke taal, de meest voorkomende oorzaken van een lage score en de concrete oplossingen per metric.

BJ
Bert-Jan Russchen
SEO & Automatisering Specialist

Wat zijn Core Web Vitals?

Core Web Vitals zijn drie metrics die Google gebruikt om de paginaervaring van een website te meten. Ze zijn in 2021 officieel onderdeel geworden van het ranking-algoritme, wat ze direct relevant maakt voor iedereen die wil scoren in Google. Lees ook hoe dit past in het bredere beeld van Google's ranking-algoritme.

De drie metrics zijn:

  • LCP, Largest Contentful Paint: hoe snel laadt het grootste zichtbare element?
  • INP, Interaction to Next Paint: hoe snel reageert de pagina op interactie?
  • CLS, Cumulative Layout Shift: springen elementen onverwacht in de layout?

Per metric gelden drempelwaarden: groen (goed), oranje (verbetering nodig) en rood (slecht). Google weegt de field data, echte gebruiksdata, zwaarder dan lab-data bij het bepalen van je ranking.

LCP, Largest Contentful Paint

LCP meet hoe snel het grootste zichtbare element op de pagina geladen is. Dat is vaak een hero-afbeelding, een grote tekstregel of een videothumbnail. De drempelwaarden: goed is onder 2,5 seconden; slecht is boven 4 seconden.

Meest voorkomende oorzaken van een slechte LCP:

  • Grote, niet-gecomprimeerde afbeeldingen (de meest voorkomende oorzaak)
  • Trage serverresponstijd (TTFB boven 600ms)
  • Render-blocking CSS of JavaScript
  • Geen gebruik van een CDN

Oplossingen:

  • Comprimeer afbeeldingen en gebruik WebP of AVIF formaat
  • Voeg fetchpriority="high" toe aan de hero-afbeelding
  • Gebruik een snelle hosting/CDN (Cloudflare, Netlify, Vercel)
  • Activeer server-side caching

INP, Interaction to Next Paint

INP verving in 2024 FID (First Input Delay) als maatstaf voor interactiviteit. Waar FID alleen de eerste interactie mat, meet INP de responstijd van alle interacties gedurende het hele bezoek. Goed is onder 200ms; slecht is boven 500ms.

INP-problemen zijn vaker aanwezig op JavaScript-zwaar websites, denk aan complexe React-applicaties, grote plugins of veel third-party scripts (chatbots, analytics, advertenties).

Oplossingen:

  • Vermijd lange JavaScript-taken (Long Tasks van >50ms)
  • Gebruik code splitting om JavaScript gefaseerd te laden
  • Laad third-party scripts asynchroon of deferred
  • Verwijder ongebruikte JavaScript en plugins

CLS, Cumulative Layout Shift

CLS meet hoe stabiel de lay-out van een pagina is tijdens het laden. Als afbeeldingen, advertenties of fonts onverwacht verschuiven nadat de pagina al zichtbaar is, dan verschuift ook de inhoud die je al las, frustrerend en schaadelijk voor je score. Goed is onder 0,1; slecht is boven 0,25.

Meest voorkomende oorzaken:

  • Afbeeldingen zonder width en height attributen
  • Advertenties of embeds die groter laden dan gereserveerde ruimte
  • Web fonts die later laden (FOIT/FOUT)
  • Dynamisch ingevoegde content bovenaan de pagina

Oplossingen:

  • Voeg altijd width en height attributen toe aan <img>-tags
  • Reserveer ruimte voor advertenties met min-height
  • Gebruik font-display: swap in je CSS voor custom fonts
  • Voorkom het injecteren van content bovenaan na het laden

Hoe meet je Core Web Vitals?

De beste tools om je scores te controleren:

  • PageSpeed Insights: toont zowel lab-data als field data per URL
  • Google Search Console: Core Web Vitals-rapport voor de hele site, inclusief trends
  • Chrome DevTools: Performance-tab voor gedetailleerde analyse per laadstap
  • WebPageTest: uitgebreide filmstrip-analyse voor waterfall-diagrammen

Field data vs Lab data. PageSpeed Insights toont twee soorten data: lab-data (gemeten in gecontroleerde omstandigheid) en field data (echte gebruiksdata uit de Chrome User Experience Report). Google gebruikt field data voor ranking. Als je lab-data groen is maar field data rood, dan is er een probleem op échte apparaten van echte gebruikers.

Invloed op SEO

Core Web Vitals zijn één onderdeel van Google's Page Experience-signaal, dat ook HTTPS, mobile-friendliness en het ontbreken van intrusive interstitials omvat. Het is geen dominante rankingfactor, relevante content weegt zwaarder, maar bij gelijke content geeft een goede paginaervaring de doorslag.

De indirecte impact is misschien nog groter: een snellere website leidt tot minder bounces, meer paginaweergaven en hogere conversieratio's. Dat verbetert je user engagement-signalen, die Google ook meeneemt in de ranking. Meer technische SEO-context vind je in de technische SEO-checklist.
