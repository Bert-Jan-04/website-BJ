Technische SEO: de checklist voor een foutloze website

Van canonical tags en sitemap tot Core Web Vitals, de technische basis die elke website op orde moet hebben voordat je aan content begint.

BJ
Bert-Jan Russchen
SEO & Automatisering Specialist

Wat is technische SEO?

Technische SEO is de fundering van je hele SEO-strategie. Het gaat om alles wat Google nodig heeft om je website te kunnen crawlen, begrijpen en indexeren, los van de content zelf. Zelfs de beste content scoort niet als Google die pagina niet kan vinden of correct kan lezen.

Technische problemen zijn ook verraderlijk: ze zijn vaak onzichtbaar voor de bezoeker, maar hebben grote gevolgen voor je rankings. Met deze checklist loop je de belangrijkste punten af die ik bij elke SEO-audit als eerste controleer. Wil je weten hoe Google vervolgens rangschikt? Lees dan ook hoe het ranking-algoritme werkt.

Crawling en indexering begrijpen

Googlebot bezoekt je website via links. Het volgt elke link die het tegenkomt en brengt zo de structuur van je site in kaart. Pagina's die geen inkomende links hebben (zogenaamde orphan pages) worden zelden gecrawld.

Wat je moet controleren:

  • Staat er geen noindex-tag op pagina's die wél geïndexeerd moeten worden?
  • Blokkeert je robots.txt geen belangrijke pagina's of mappen?
  • Open Google Search Console en controleer het Coverage-rapport op fouten en waarschuwingen
  • Zijn er pagina's met nofollow op interne links die dat niet zouden moeten hebben?

Google Search Console is je beste vriend. Dit gratis hulpmiddel van Google laat precies zien welke pagina's geïndexeerd zijn, welke fouten er zijn, en hoe je site presteert in zoekresultaten. Als je het nog niet hebt ingesteld: doe dat vandaag nog.

Sitemap.xml en robots.txt

Een sitemap.xml is een bestand dat Google vertelt welke pagina's op je website bestaan en wanneer ze voor het laatst zijn bijgewerkt. Het is geen garantie dat Google die pagina's indexeert, maar het helpt wel, zeker voor nieuwe websites of grote sites met veel pagina's.

Je sitemap is meestal bereikbaar via jouwdomein.nl/sitemap.xml. Dien hem in via Google Search Console onder het menu "Sitemaps". Verwijder verouderde pagina's en 404-fouten uit je sitemap.

Het robots.txt-bestand staat in de root van je website en vertelt crawlers welke pagina's ze mogen bezoeken. Een veelgemaakte fout is het per ongeluk blokkeren van CSS- of JavaScript-bestanden, waardoor Google je website niet correct kan renderen.

Canonical tags voorkomen duplicate content

Duplicate content ontstaat wanneer dezelfde (of vrijwel dezelfde) content op meerdere URL's bereikbaar is. Dit verdeelt je "linkwaarde" over meerdere pagina's en verwarrt Google over welke versie het moet ranken.

Veelvoorkomende oorzaken:

  • HTTP én HTTPS versies van je site die beide bereikbaar zijn
  • www en non-www versies
  • URL-parameters van filters of tracking (?utm_source=...)
  • Pagineringen zonder canonical

De oplossing is een <link rel="canonical" href="...">-tag in de <head> van je pagina, die verwijst naar de voorkeursversie. Zorg ook voor een 301-redirect van de niet-canonieke versies.

Paginasnelheid en Core Web Vitals

Snelheid is een directe rankingfactor. Core Web Vitals, LCP, INP en CLS, zijn de drie metrics die Google gebruikt om paginaervaring te meten. Een slechte score heeft meetbare gevolgen voor je rankings.

De meest impactvolle verbeteringen voor snelheid:

  • Comprimeer afbeeldingen en gebruik moderne formaten (WebP, AVIF)
  • Laad JavaScript deferred of asynchroon
  • Gebruik een CDN (Content Delivery Network)
  • Activeer browsercaching en Gzip-compressie op de server
  • Verwijder ongebruikte CSS en JavaScript

Mobile-first indexing

Sinds 2023 indexeert Google uitsluitend op basis van de mobiele versie van je website. Dit betekent dat content die alleen op de desktopversie staat, niet meegenomen wordt in de ranking.

Zorg voor een volledig responsive ontwerp waarbij de mobiele versie dezelfde content bevat als de desktopversie. Test je website in Chrome DevTools (toggle device toolbar) en via Google's Mobile-Friendly Test.

Voor technische details over hoe je website is gebouwd, kan een afweging tussen WordPress en maatwerk relevant zijn, het platform bepaalt mede hoeveel controle je hebt over mobiele optimalisatie.

Structured data en Schema markup

Structured data is code die je aan je HTML toevoegt om Google extra context te geven over wat een pagina beschrijft. Het kan leiden tot rich results: zoekresultaten met stervormige beoordelingen, FAQ-uitklappers, receptinformatie of evenementdata.

Meest gebruikte schema-typen voor MKB-websites:

  • LocalBusiness: naam, adres, telefoonnummer, openingstijden
  • FAQ: veelgestelde vragen die uitklappen in zoekresultaten
  • Article: voor kennisartikelen en blogposts
  • BreadcrumbList: navigatiepad in de zoekresultaten

Gebruik de Schema.org validator om je markup te controleren op fouten.

Een gezonde technische basis is de voorwaarde voor alles wat daarna komt. Wil je dieper ingaan op snelheid? Lees het artikel over Core Web Vitals. Voor de volgende stap in je SEO-strategie, bekijk hoe het ranking-algoritme werkt.
