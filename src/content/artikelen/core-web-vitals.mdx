## Wat zijn Core Web Vitals?
<p>Core Web Vitals zijn drie metrics die Google gebruikt om de paginaervaring van een website te meten. Ze zijn in 2021 officieel onderdeel geworden van het ranking-algoritme, wat ze direct relevant maakt voor iedereen die wil scoren in Google. Lees ook hoe dit past in het bredere beeld van <a href="/artikelen/seo-ranking-algoritme">Google's ranking-algoritme</a>.</p>
<p>De drie metrics zijn:</p>
<ul>
  <li><strong>LCP</strong>, Largest Contentful Paint: hoe snel laadt het grootste zichtbare element?</li>
  <li><strong>INP</strong>, Interaction to Next Paint: hoe snel reageert de pagina op interactie?</li>
  <li><strong>CLS</strong>, Cumulative Layout Shift: springen elementen onverwacht in de layout?</li>
</ul>
<p>Per metric gelden drempelwaarden: groen (goed), oranje (verbetering nodig) en rood (slecht). Google weegt de field data, echte gebruiksdata, zwaarder dan lab-data bij het bepalen van je ranking.</p>

## LCP, Largest Contentful Paint
<p>LCP meet hoe snel het grootste zichtbare element op de pagina geladen is. Dat is vaak een hero-afbeelding, een grote tekstregel of een videothumbnail. De drempelwaarden: goed is onder 2,5 seconden; slecht is boven 4 seconden.</p>
<p><strong>Meest voorkomende oorzaken van een slechte LCP:</strong></p>
<ul>
  <li>Grote, niet-gecomprimeerde afbeeldingen (de meest voorkomende oorzaak)</li>
  <li>Trage serverresponstijd (TTFB boven 600ms)</li>
  <li>Render-blocking CSS of JavaScript</li>
  <li>Geen gebruik van een CDN</li>
</ul>
<p><strong>Oplossingen:</strong></p>
<ul>
  <li>Comprimeer afbeeldingen en gebruik WebP of AVIF formaat</li>
  <li>Voeg <code>fetchpriority="high"</code> toe aan de hero-afbeelding</li>
  <li>Gebruik een snelle hosting/CDN (Cloudflare, Netlify, Vercel)</li>
  <li>Activeer server-side caching</li>
</ul>

## INP, Interaction to Next Paint
<p>INP verving in 2024 FID (First Input Delay) als maatstaf voor interactiviteit. Waar FID alleen de eerste interactie mat, meet INP de responstijd van alle interacties gedurende het hele bezoek. Goed is onder 200ms; slecht is boven 500ms.</p>
<p>INP-problemen zijn vaker aanwezig op JavaScript-zwaar websites, denk aan complexe React-applicaties, grote plugins of veel third-party scripts (chatbots, analytics, advertenties).</p>
<p><strong>Oplossingen:</strong></p>
<ul>
  <li>Vermijd lange JavaScript-taken (Long Tasks van &gt;50ms)</li>
  <li>Gebruik code splitting om JavaScript gefaseerd te laden</li>
  <li>Laad third-party scripts asynchroon of deferred</li>
  <li>Verwijder ongebruikte JavaScript en plugins</li>
</ul>

## CLS, Cumulative Layout Shift
<p>CLS meet hoe stabiel de lay-out van een pagina is tijdens het laden. Als afbeeldingen, advertenties of fonts onverwacht verschuiven nadat de pagina al zichtbaar is, dan verschuift ook de inhoud die je al las, frustrerend en schaadelijk voor je score. Goed is onder 0,1; slecht is boven 0,25.</p>
<p><strong>Meest voorkomende oorzaken:</strong></p>
<ul>
  <li>Afbeeldingen zonder width en height attributen</li>
  <li>Advertenties of embeds die groter laden dan gereserveerde ruimte</li>
  <li>Web fonts die later laden (FOIT/FOUT)</li>
  <li>Dynamisch ingevoegde content bovenaan de pagina</li>
</ul>
<p><strong>Oplossingen:</strong></p>
<ul>
  <li>Voeg altijd <code>width</code> en <code>height</code> attributen toe aan <code>&lt;img&gt;</code>-tags</li>
  <li>Reserveer ruimte voor advertenties met <code>min-height</code></li>
  <li>Gebruik <code>font-display: swap</code> in je CSS voor custom fonts</li>
  <li>Voorkom het injecteren van content bovenaan na het laden</li>
</ul>

## Hoe meet je Core Web Vitals?
<p>De beste tools om je scores te controleren:</p>
<ul>
  <li><strong>PageSpeed Insights</strong>: toont zowel lab-data als field data per URL</li>
  <li><strong>Google Search Console</strong>: Core Web Vitals-rapport voor de hele site, inclusief trends</li>
  <li><strong>Chrome DevTools</strong>: Performance-tab voor gedetailleerde analyse per laadstap</li>
  <li><strong>WebPageTest</strong>: uitgebreide filmstrip-analyse voor waterfall-diagrammen</li>
</ul>

<div class="artikel-callout artikel-callout--purple">
  <p><strong>Field data vs Lab data.</strong> PageSpeed Insights toont twee soorten data: lab-data (gemeten in gecontroleerde omstandigheid) en field data (echte gebruiksdata uit de Chrome User Experience Report). Google gebruikt field data voor ranking. Als je lab-data groen is maar field data rood, dan is er een probleem op échte apparaten van echte gebruikers.</p>
</div>

## Invloed op SEO
<p>Core Web Vitals zijn één onderdeel van Google's Page Experience-signaal, dat ook HTTPS, mobile-friendliness en het ontbreken van intrusive interstitials omvat. Het is geen dominante rankingfactor, relevante content weegt zwaarder, maar bij gelijke content geeft een goede paginaervaring de doorslag.</p>
<p>De indirecte impact is misschien nog groter: een snellere website leidt tot minder bounces, meer paginaweergaven en hogere conversieratio's. Dat verbetert je user engagement-signalen, die Google ook meeneemt in de ranking. Meer technische SEO-context vind je in de <a href="/artikelen/technische-seo-checklist">technische SEO-checklist</a>. Voor advies over welk platform het makkelijkst te optimaliseren is, lees het artikel over <a href="/artikelen/wordpress-vs-maatwerk">WordPress vs. maatwerk</a>.</p>
